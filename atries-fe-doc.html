ATRIES Component Library

Arcadis Tailings Risk Information & Engineering System — A collection of accessible, production-ready UI components built with design tokens and best practices.

Overview

ATRIES provides a complete set of UI components for building tailings-management applications. Built with plain HTML, CSS, and JavaScript, every component follows accessibility standards and uses a consistent design token system.

  • Design tokens — Colors, spacing, typography from a centralized token system
  • Accessible by default — ARIA roles, keyboard navigation, focus management
  • Production ready — Ready-to-use code that can be directly implemented in Angular framework

Accessibility

ATRIES components are built with accessibility as a core principle. The application achieves a 100% accessibility score in Lighthouse audits.

Key Features

  • Semantic HTML with proper landmark regions
  • ARIA roles and attributes for screen readers
  • Full keyboard navigation support
  • Visible focus indicators on all interactive elements
  • Skip link to main content
  • Dropdown menus with proper menu/menuitem roles
  • Form controls with accessible labels

Components

Buttons

Three button variants: Dark (primary), Outline Dark (secondary), and Flat (tertiary).

Forms

Search fields, checkboxes, radio buttons, and switches with consistent styling.

Navigation

Top navigation bar, collapsible sidebar with accordion groups, breadcrumbs, and tabs.

Data Display

Tables with sortable columns, pagination, and column visibility controls.

Menus

Dropdown menus with keyboard navigation and proper ARIA implementation.

Cards

Dashboard cards with icons, titles, and descriptions for navigation.

Getting Started

For Developers

  1. Browse the component gallery to see all available components
  2. View the dashboard demo to see the full layout in action
  3. Copy HTML markup and CSS classes from component pages
  4. Use design tokens from luma-tokens/ for consistent styling

Design Tokens

All components use centralized design tokens for:

  • Colors (text, surface, border)
  • Spacing (0.25rem to 12rem scale)
  • Typography (font sizes, weights, line heights)
  • Border radius and widths

File Structure

  • luma-tokens/ — Design token CSS variables
  • css/ — Component stylesheets
  • js/ — Component interactions
  • templates/ — Component demo pages
  • images/ — Icons and assets
