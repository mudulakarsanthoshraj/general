Project Overview

BDB UI External is a React-based web application that provides external users with access to construction industry data, market indicators, and project management tools. Built with modern technologies and designed for scalability and maintainability.

What is BDB?

BDB (likely "Bouw Database" - Construction Database) is a platform providing:

  • Market Indicators: Real-time construction industry trends and analytics
  • Monthly Figures: Historical and forecast construction cost indices
  • Aspect Ratios: Detailed cost breakdowns by construction categories
  • Parametric Elements: Construction element cost analysis
  • Reference Projects: Project-based budget and cost tracking
  • Flexible Indices: Customizable index calculations
Key Application Features
  • Multi-tenant architecture with company-based access control
  • Role-based permissions (Admin, Standard User)
  • Subscription-based feature access
  • Comprehensive data export capabilities (Excel, PDF)
  • Internationalized interface (Dutch/English support)
  • Customizable dashboard with drag-and-drop tiles
  • Session timeout management and security controls

Key Architectural Patterns

1. Component-Based Architecture

The application follows React best practices with functional components and hooks. Components are organized by feature and reusability.

  • Pages: Top-level route components in src/pages/
  • Components: Reusable UI components in src/components/
  • Layouts: Structural wrappers in src/layout/

2. Service Layer Pattern

All API calls are abstracted into service modules. Each service corresponds to a backend resource or feature area. 

// Example: MonthlyFiguresService.ts
import http from "./common/HttpCommon.ts";

const getMonthlyFigures = async (id, isForecastIncluded, referenceDates) => {
  const response = await http.post(`ReportForecastFigures/${id}`, referenceDates);
  return response.data;
};

export default { getMonthlyFigures };

3. Authentication & Authorization

Uses Microsoft Authentication Library (MSAL) for Azure B2C integration:

  • Token-based authentication with automatic refresh
  • Session storage for tokens and user data
  • HTTP interceptors add Bearer tokens to requests
  • Role-based route protection via PrivateRoute

4. State Management

Global State (Redux)

  • Snackbar/notification state
  • Managed via Redux Toolkit
  • Used sparingly for cross-cutting concerns

Local State (Context + Hooks)

  • App-level state (clientId)
  • Component-specific state
  • Custom hooks for data fetching

5. Data Models

TypeScript interfaces define all data structures. Models are organized by feature:

  • IMonthlyFigures.ts - Monthly construction figures data
  • IMarketIndicator.ts - Market indicator data structures
  • IAspectRatio.ts - Cost breakdown and analysis data
  • IProfile.ts - User profile and company information
  • IProject.ts - Reference project data

Authentication (Azure B2C)

The application uses Microsoft Azure AD B2C for user authentication and authorization.

Authentication Flow

User visits app → useAuth hook → MSAL acquireToken → Azure B2C
Token received → Stored in sessionStorage → Profile fetched → App ready

Configuration (auth-config.ts)

export const msalConfig = {
  auth: {
    clientId: "27848361-a66e-4158-856c-8414a88f7053",  // From env
    authority: `https://${tenant}.b2clogin.com/...`,
    knownAuthorities: ["arcadisb2cdev.b2clogin.com"],
    redirectUri: "https://devbdb.arcadis.com/dashboard",
  },
  cache: {
    cacheLocation: "sessionStorage",
    storeAuthStateInCookie: false,
  }
};

Authentication Hook (useAuth)

Custom hook in src/hooks/auth.hook.ts manages token acquisition and caching:

  • Checks for valid token in sessionStorage first
  • If expired, attempts silent token renewal via MSAL
  • Falls back to interactive login if silent renewal fails
  • Stores token and expiration time in sessionStorage
Token Lifecycle
  • Acquisition: acquireTokenSilent() or acquireTokenPopup()
  • Storage: appStorage.setAccessToken()
  • Injection: Axios interceptor adds Bearer token to all requests
  • Refresh: Automatic via authOperations.getRefreshToken()
  • Expiration: Session timeout dialog warns before expiry

App.tsx Authentication Flow

const { accessToken, loading } = useAuth({ scopes: [scopes] });

useEffect(() => {
  if (!loading) {
    if (accessToken) {
      appStorage.setAccessToken(accessToken);
      
      // Fetch user profile and company info
      ProfileService.getProfile()
        .then((profile) => {
          appStorage.setClientName(profile.companyName);
          appStorage.setSelectedClientId(profile.companyId);
        });
    } else {
      // No token - clear storage
      appStorage.removeTokenFromStorage();
    }
  }
}, [accessToken, loading]);

HTTP Request Authorization

HttpCommon.ts configures Axios with token injection: 

instance.interceptors.request.use(
  async (config) => {
    await authOperations.getRefreshToken();  // Ensure fresh token
    const token = appStorage.getAccessToken();
    
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => Promise.reject(error)
);

Session Management

Session Timeout

SessionTimeoutDialog component monitors token expiration and warns users before session ends. Users can extend their session or will be logged out automatically.

Services & API Integration

API service layer architecture and how the application communicates with the backend.

Service Layer Architecture

All backend communication is abstracted into service modules located in src/services/. Each service handles a specific domain or feature area.

HttpCommon.ts - Base Configuration

import axios from 'axios';

const instance = axios.create({
  baseURL: import.meta.env.VITE_REACT_APP_API_URL,  // e.g., https://devbdb.arcadis.com/api/bdbapi/api/
  headers: {
    Accept: 'application/json',
    'Content-Type': 'application/json',
  },
});

// Request interceptor - adds auth token
instance.interceptors.request.use(
  async (config) => {
    await authOperations.getRefreshToken();
    const token = appStorage.getAccessToken();
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  }
);

export default instance;

Service Pattern Example

// MonthlyFiguresService.ts
import http from "./common/HttpCommon.ts";
import { MonthlyFiguresResponseData } from "../models/IMonthlyFigures.ts";

const getMonthlyFigures = async (
  id: number,
  isForecastIncluded: boolean,
  referenceDates: string[]
): Promise<MonthlyFiguresResponseData> => {
  const response = await http.post<any>(
    `ReportForecastFigures/${id}?isForecastIncluded=${isForecastIncluded}`,
    referenceDates
  );
  return response.data;
};

const MonthlyFiguresService = {
  getMonthlyFigures,
  getMonthlyFiguresReport,
};

export default MonthlyFiguresService;

API Response Patterns

Common Response Structures
  • Data Lists: { rows: T[], totalCount?: number }
  • Single Entity: Entity object directly
  • File Downloads: Blob with responseType: 'blob'
  • Metadata: { lastUpdated: string, data: T }

Error Handling

Services throw errors that are caught by calling components. Common error handling pattern: 

try {
  const data = await MonthlyFiguresService.getMonthlyFigures(id, true, dates);
  setData(data);
} catch (error) {
  console.error("Failed to fetch monthly figures:", error);
  // Show snackbar notification
  dispatch(showSnackbar({ message: "Error loading data", severity: "error" }));
}

State Management

How state is managed across the application using Context API and Redux.

State Architecture

Global State (Redux)

  • Snackbar Notifications
  • Managed by Redux Toolkit
  • Single centralized store
  • Used for cross-component communication

App Context (React Context)

  • Client ID selection
  • Lightweight app-level state
  • Avoids prop drilling
  • Used for session-scope data

App State Context

// appState.context.tsx
export type AppStateContextType = {
  clientId: number;
  setClientId: (clientId: number) => void;
};

export const AppStateContext = createContext<AppStateContextType>({
  clientId: 0,
  setClientId: () => {},
});

// Usage in App.tsx
const [clientId, setClientId] = useState<number>(0);
const memoizedValue = useMemo(() => ({ clientId, setClientId }), [clientId]);

return (
  <AppStateContext.Provider value={memoizedValue}>
    {children}
  </AppStateContext.Provider>
);

Redux Store (Notifications)

// SnackbarNotification/store.ts
import { configureStore, createSlice } from "@reduxjs/toolkit";

const snackbarSlice = createSlice({
  name: "snackbar",
  initialState: { open: false, message: "", severity: "info" },
  reducers: {
    showSnackbar: (state, action) => ({
      ...state,
      open: true,
      message: action.payload.message,
      severity: action.payload.severity,
    }),
    hideSnackbar: (state) => ({ ...state, open: false }),
  },
});

const store = configureStore({
  reducer: { snackbar: snackbarSlice.reducer },
});

export default store;

Troubleshooting

Common issues and their solutions.

Development Server Issues

Port 3000 Already in Use

Solution

Kill the process using port 3000:

lsof -ti:3000 | xargs kill -9

Or change the port in vite.config.ts

Module Not Found Errors

Solution
  1. Delete node_modules/ and package-lock.json
  2. Run npm install again
  3. Restart the dev server

Authentication Issues

Infinite Login Loop

Possible Causes
  • Token expiration time not set correctly
  • Azure B2C configuration mismatch
  • Redirect URL not matching environment
  • SessionStorage being cleared unexpectedly

Solution:

  • Check VITE_REACT_APP_REDIRECT_URL matches current URL
  • Verify Azure B2C app registration settings
  • Clear browser cache and sessionStorage
  • Check console for MSAL errors

401 Unauthorized on API Calls

// Check if token is being sent
// Open browser DevTools → Network → Headers
# Look for: Authorization: Bearer [token]

// Debug token acquisition
console.log(appStorage.getAccessToken());
console.log(appStorage.getExpirationTime());

Build Issues

TypeScript Compilation Errors

Solution
  • Run npm run lint to see all type errors
  • Check tsconfig.json for strict mode settings
  • Ensure all imports have correct types
  • Use // @ts-expect-error only as last resort

Environment Variables Not Loading

Remember
  • Vite env vars must start with VITE_
  • Changes require server restart
  • Check you're using import.meta.env not process.env
  • Verify the correct .env.* file for your build command

Data Grid Issues (AG Grid)

License Warning in Console

AG Grid Enterprise requires a license. Check package.json for @mui/x-license package.

Grid Not Rendering

  • Ensure parent container has explicit height
  • Check data format matches column definitions
  • Verify AG Grid styles are imported

Translation Issues

Translations Not Loading

Checklist
  • Check network tab for i18n API calls
  • Verify solution ID and code in i18n.ts
  • Ensure translations exist on platform
  • Check browser console for i18next errors

Performance Issues

Slow Page Load

  • Check React DevTools Profiler for slow components
  • Look for unnecessary re-renders
  • Verify data fetching isn't happening in render
  • Use useMemo and useCallback appropriately

Large Bundle Size

  • Run npm run build and check dist/ folder size
  • Consider lazy loading heavy components
  • Check for duplicate dependencies
  • Use tree-shakeable imports (e.g., import { Button } from "@mui/material")
