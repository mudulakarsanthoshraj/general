The Ask

We’d like approval for a single, low-cost API key so our team can use an approved low-cost AI model inside our own apps, internally. We’re not tied to one vendor — any of today’s cheap models works.

What we’re asking management to approve

  1. One approved API key to one low-cost AI model (whichever cheap, approved model fits best), used only internally.
  2. A small monthly spending limit on that key — around $25–$50, set by leadership. Expected real spend is about $1–$30 a month; the limit is a safety ceiling, not a fixed bill.

What this is

  • One approved model, one key, used internally.
  • Small, watched usage for learning and trying ideas.
  • A reusable setup that future approved ideas can share.

What this is not

  • Not a key for every engineer or laptop.
  • Not a consumer AI subscription (chat assistants or coding copilots) — those are separate decisions.
In one line
For a small, capped monthly cost — with no private data shared and all security rules followed — we get hands-on access to use AI safely inside our own apps.

What changes for the company and the team once we can use AI safely in our own apps.

With an approved API key

  • The team can try real AI ideas directly in our apps.

Bottom line
A small, safe start lets us use AI the smart way — learn first, see what works, then grow — instead of guessing from theory or vendor demos.

How It Works in Real Time

A “real-time” AI feature means the user does something in the product and gets a useful answer in a second or two. Every one follows the same simple loop — and it all runs internally, using the approved key.

User does something → Internal app adds our data + the key → AI model → Answer shown in the app

Real examples in our products

POC done — the AI design-system assistant
We’ve completed a working POC. The AI design-system assistant was built inside our design-system docs and answers real questions like “How do I use the Button component?” — with working code grounded on our own docs, not guesses.
“How do I use the Button component?” → Find matching design-system docs → AI model → Answer + code
How the AI Assistant works, step by step
  1. The user types a question in plain words.
  2. Our code finds the most relevant product doc snippets and adds them to the prompt (so answers stay grounded in our real docs).
  3. It sends the question + those snippets to the model using the standard chat API.
  4. The model replies, and the answer (with code) shows up in the chat window.

Because it speaks the standard chat API that most vendors share, the very same code runs on a free local model now and on any approved low-cost model later — we only change one setting, no rewrite or vendor lock-in.

Another in-app help chatbot
The same pattern works in any app: the user asks a question in plain words; the app finds the matching help docs and the model answers with a link to the source.
“How do I export a report?” → Find matching docs → AI model → Answer + link

What this means in real money
A typical request on any of these low-cost models costs well under a cent. Even at 10,000 requests a month that’s only a few dollars — far below the proposed limit. That’s why we expect real spend of just $1–$30 a month, with the cap as a safety ceiling.
Right tool for the job
Low-cost models are great for chatbots, finding and organizing data, and summarizing — but not for writing complex code or hard problem-solving. We use a cheap model for everyday tasks, and only consider a pricier model if a specific need comes up.

How It Stays Safe

Simple rules that keep the spend, the data, and the answers under control.

Rule How it works
One approved model We start with one approved low-cost model. Any change of model goes through Security review.
Monthly spending limit The key has a fixed monthly cap and stops on its own when it’s reached.
No private data Public docs and sample data only — no personal details (names, emails, employee IDs), no private code, no passwords.
Runs only internally The key and logic stay internal, never in the browser or on a laptop.
Full record Every prompt and response is logged for 30 days so Security can review it.
